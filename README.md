## Коллекция моих наработок в области автоматизации бухгалтерских задач.

 Все проекты создаются в условиях высокой рабочей нагрузки с целью автоматизации рутинных операций бухгалтера и находятся в стадии MVP.

**Технический стек**: 
* учет ведется в SAP R3 с использованием стандартных транзакций и доступа рядового бухгалтера\клерка, без доступа к внутренним регистрам;
* Обмен электронными документами ведется через Контур.Диадок с доступом исключительно к веб-интерфейсу, без возможности использования API либо модуля "Диадок.Коннектор";
* справочники с расшифровками названий кредиторов, МВП/МВЗ, внутренним планом счетов хранятся в .xlsx файлах;
* автоматизация на Python с использованием библиотек:
	* Pandas
	* Etree
	* Zipfile
	* Xlwings
	* Openpyxl
	* Jupyter Widgets
	* Mathplotlib
* я использую как самодостаточные скрипты python, которые запускаются из командной строки, так и Jupiter Notebook для задач, где в первую очередь требуется более интерактивно работать с исходными даными и наглядно предствалять результат анализа.

Я придерживаюсь принципов бесшовной интеграции, адаптируя свои расчеты под действующий порядок документооборота без необходимости его перестроения либо получения расширенных прав доступа к информационным системам.

В скриптах, в частности в обработке GRIR, внедрено автоматическое кэширование справочника контрагентов из .xls в pickle, что существенно ускоряет выполнение обработок.

Подходы, которых сейчас не хватает и которые я планирую внедрить в эти проекты:

* использовать named tuples для названий колонок САП отчетов (сделает код нагляднее)
* добавить requirements.txt к проектам
* везде использовать отельный файл settings.py с константами для скрипта (пути к файлам, названия колонок), в скрипте делать импорт вида import config as cfg; print(cfg.PATH) как первый шаг, позже перейти на использование полноценных .env файлов
* полноценно внедрить библиотеку logging, для облегчения отладки
* добавить обработку параметров командной строки для скриптов на базе argparse
* использовать библиотеку pathlib для работу с путями

Перед публикацией некоторые фрагменты кода были отредактированы из соображений конфиденциальости.
